Taylor-Green Vortex (Convergence Study)¶

The Taylor-Green vortex is the canonical incompressible-flow benchmark with a known exact solution: a periodic decaying vortex on the square \([0, 2\pi]^2\) whose velocity and pressure can be written down in closed form. The script examples/fluid/taylor_green/taylor_green.py uses it to drive a quantitative h-convergence study of the transient Navier-Stokes solver — the only example in Fluid Mechanics whose purpose is verification rather than visualization.

Exact solution¶

For viscosity \(\nu\) and time \(t\),

\[\begin{split}u(x, y, t) &= -\cos(x)\, \sin(y)\, e^{-2\nu t}, \\ v(x, y, t) &= \phantom{-}\sin(x)\, \cos(y)\, e^{-2\nu t}, \\ p(x, y, t) &= -\tfrac14 \bigl(\cos(2x) + \cos(2y)\bigr)\, e^{-4\nu t}.\end{split}\]

The velocity decays exponentially with rate \(2\nu\) (twice the kinematic viscosity); the pressure decays at rate \(4\nu\). Substitution verifies that \((u, v, p)\) satisfies the incompressible Navier-Stokes equations exactly.

The script imposes Dirichlet velocities on every boundary node equal to the exact solution at the current time. This sidesteps the need for periodic boundaries and isolates the spatial / temporal discretization error.

Solver¶

The transient NavierStokesTransientAssembler from Cylinder Flow (Vortex Shedding) is reused verbatim — implicit Euler in time, SUPG/PSPG with adaptive \(\tau\) in space. The only changes are:

  • domain \([0, 2\pi]^2\) instead of the cylinder channel,

  • Dirichlet velocity from the exact solution rather than a parabolic inlet,

  • every step writes \(L^2\) errors against the analytical velocity and pressure to a results file.

Listing 19 examples/fluid/taylor_green/taylor_green.py (essence)¶
def exact_velocity(points, t, nu):
    x, y = points[:, 0], points[:, 1]
    decay = math.exp(-2.0 * nu * t)
    u = -torch.cos(x) * torch.sin(y) * decay
    v =  torch.sin(x) * torch.cos(y) * decay
    return torch.stack([u, v], dim=1)

    mesh = Mesh.gen_rectangle(left=0, right=2*math.pi,

h-convergence¶

The script sweeps the mesh size \(h = 2\pi / n_\text{grid}\) through a list of values and records the \(L^2\) velocity and pressure errors at a fixed final time. For P1-P1 elements with SUPG/PSPG one expects:

  • \(\|\mathbf{u}_h - \mathbf{u}\|_{L^2} = \mathcal{O}(h^2)\)

  • \(\|p_h - p\|_{L^2} = \mathcal{O}(h)\)

The console output is a table:

   n_grid     h       err_u_L2    rate_u    err_p_L2    rate_p
---------------------------------------------------------------
     10    0.628    1.3e-1                  4.5e-1
     20    0.314    3.4e-2     1.9          2.3e-1     1.0
     40    0.157    8.7e-3     2.0          1.2e-1     1.0
     80    0.079    2.2e-3     2.0          5.8e-2     1.0

(numbers approximate, will vary slightly with the chosen \(\nu\) and \(\Delta t\)). The plot taylor_green_convergence.png shows the same data on a log-log axis with reference \(h^2\) and \(h^1\) slopes.

If the observed rates differ from the expected \(\mathcal{O}(h^2)\) / \(\mathcal{O}(h)\), something in the solver is wrong — most often a subtle bug in the stabilization or in the boundary-condition imposition.

Mass-weighted error norm¶

The script computes the discrete \(L^2\) norm via the mass matrix:

\[\|e_h\|_{L^2}^2 \;=\; e_h^T\, M\, e_h \;=\; \sum_K \int_K e_h^2 \,\mathrm{d}\Omega,\]

assembled by MassElementAssembler. This is the right norm for FEM error analysis (it integrates over the elements, weighted by the basis), unlike the simple Euclidean norm of the nodal error vector.

Output¶

  • Console table — convergence rates at every refinement step.

  • ``taylor_green_convergence.png`` — log-log error plot vs mesh size with reference slopes.

  • ``taylor_green_results.png`` — three-panel snapshot of the finest mesh: speed, pressure, velocity-error magnitude.

  • ``taylor_green.mp4`` (optional) — animation of the decaying vortex.

Running it¶

cd examples/fluid/taylor_green
python taylor_green.py      # writes convergence + results pngs

What’s next¶

  • Lid-Driven Cavity and Cylinder Flow (Vortex Shedding) — the qualitative NS solvers whose convergence this example verifies.

  • Elements and Quadrature — switch to higher-order elements (order=2) and the expected rates shift to \(h^3\) / \(h^2\).
